01 · Структура

Информационная архитектура и навигация

Принцип: панель — для контроля, не для рутины. Рутина живёт в WhatsApp. Поэтому навигация — 5 разделов, без вложенных меню. Главный экран — «Диалоги»: администратор проводит там 90% времени.

1 · Диалоги
Живая лента · перехват · фильтры: требует внимания / все
2 · Календарь
День/неделя по врачам · создание записи вручную
3 · Пациенты
Мини-CRM · карточка · история визитов и диалогов
4 · Айым (AI)
База знаний · тон общения · тест-чат · журнал действий
5 · Отчёты
Деньги от AI · потерянные лиды · неявки

Вне навигации: настройки организации и биллинг (меню аватара), онбординг-мастер (одноразовый), нотификации (колокольчик: эскалации, сбои каналов, лимиты тарифа).

04 · Потоки

Ключевые пользовательские потоки

Поток Шаги (→) и точки выхода
Регистрация + организацияЛендинг → e-mail/Google → magic link → создание клиники (S-03) → онбординг-мастер. Без карты. Прерывание — прогресс сохраняется, возврат по ссылке из письма.
Загрузка знанийФайл/фото/текст → парсинг с прогрессом → предпросмотр «услуга—цена» → правка инлайн → публикация версии. Ошибка парсинга → строки с проблемами подсвечены, остальное сохранено.
AI-чат (пациент)Сообщение → ответ ≤ 5 с → [вопрос вне базы → эскалация + «уточню у администратора»] → предложение записи → подтверждение → карточка в CRM. Все ветки заканчиваются либо записью, либо задачей человеку — «тихих» концов нет.
Запись (появление в календаре)AI резервирует слот (блокировка 5 мин) → подтверждение пациента → запись в календаре + уведомление админу → напоминания −24 ч / −2 ч → визит / перенос / неявка (фиксируется в CRM).
CRM-потокНовый контакт создаётся автоматически при первом сообщении → AI заполняет имя/интерес → статусы: лид → записан → был → потерян (с причиной). Ручное редактирование — только поверх, автоданные не затираются.
БиллингТриал 14 дн → напоминания за 3 дн и 1 дн с цифрой «AI принёс ₸N» → оплата картой/Kaspi → автопродление. Неуплата → 7 дней read-only (AI отвечает, панель заморожена) → пауза. Данные не удаляются 90 дней.
05 · Состояния

Error · Empty · Loading

Error
Ошибки говорят, что делать, а не что сломалось: «WhatsApp отключился — нажмите, чтобы переподключить». Критичные (канал упал, лимит диалогов) — баннер поверх всех экранов + пуш владельцу. Ошибка AI никогда не видна пациенту — только фолбэк-ответ.
Empty
Пустые состояния продают следующую настройку: пустые «Диалоги» → «Отправьте себе сообщение — проверьте Айым»; пустые «Отчёты» → «Первый отчёт появится в понедельник». Никаких пустых таблиц без действия.
Loading
Скелетоны вместо спиннеров в ленте и календаре; парсинг прайса — прогресс с этапами («читаю файл → нахожу услуги → строю ответы»); оптимистичные обновления для перехвата и правок CRM.
06 · Правила

Доступность и отзывчивость

Доступность (WCAG 2.1 AA)
  • Контраст текста ≥ 4,5:1; статусы не только цветом (иконка + текст)
  • Полная клавиатурная навигация ленты и перехвата
  • Тач-цели ≥ 44px; шрифт панели ≥ 14px
  • Интерфейс на русском и казахском (переключение на уровне пользователя)
Responsive · mobile-first
  • Брейкпоинты: 360 (база) · 768 · 1200
  • Мобайл: таб-бар, один столбец, диалог — отдельный экран
  • Десктоп: трёхколоночная лента (список + диалог + карточка)
  • Календарь на мобайле — день по врачу, не неделя
